# RateWise 產品級迭代路線圖（experiment/ratewise-product-2026h2）

> 目標：把 RateWise 迭代到「頂級韓系 fintech app」產品水準（對標 Toss／Wowpass）。
> 雙軌推進：**Hotfix 軌**（小修直進 main）＋ **Experiment 軌**（大規模重構全部在本實驗分支，達標後依序 PR 回 main）。
> 治理：PM 主代理全自主指揮；每個 epic 過 Fable adversarial review 與評分卡才合併。

## 合併與推送順序

1. Hotfix 軌（H1-H3）：逐一以最小 PR 直進 main（issue 先行，gh 立案附證據）。
2. Experiment 軌：epic 內 worktree 併行 → epic review APPROVE → 併入實驗分支 → 全部 epic 完成後評分卡達標 → 依序拆 PR 回 main。
3. 全程 CI 必須綠燈；發版節奏由 changesets 管理。
